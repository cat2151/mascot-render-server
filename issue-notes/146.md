# issue cache配下のrenders/ にて、ファイル名にuser name等の上位階層pathが含まれているのをやめる #146
[issues #146](https://github.com/cat2151/mascot-render-server/issues/146)

- これまでの課題：
    - ファイル名が例えば  `Users__<user name>__AppData__Local__mascot-render-server__cache__ハッシュ文字列～__extracted__ずんだもん立ち絵素材2.3__ずんだもん立ち絵素材2.3.psd.png`
    - となっている。
    - ハッシュ文字列より前のcacheまでの部分は情報として不要である。そして長すぎるファイル名はトラブルの元である
    - user nameがファイル名に含まれてしまっているため、例えばログを共有するときにセキュリティリスクがある
- 対策：
    - ファイル名を例えば `ハッシュ文字列～__extracted__ずんだもん立ち絵素材2.3__ずんだもん立ち絵素材2.3.psd.png` に修正する
